# GitHub Pages Setup Checklist

## Repository setup

1. Create or use the GitHub repository for the project.
2. Copy the contents of `github_pages_scaffold/` into the repository root.
3. Confirm the repository contains:
   - `README.md`
   - `docs/index.md`
   - `docs/es/articulo.md`
   - `docs/en/article.md`
   - `docs/evidencia/`
   - `docs/evidence/`
   - `data/`

## GitHub Pages settings

1. Go to repository Settings.
2. Open Pages.
3. Set source to the repository branch.
4. Set the folder to `/docs`.
5. Save and wait for the Pages build.

## Pre-launch validation

Before posting publicly:

1. Open the Spanish homepage.
2. Open the Spanish article.
3. Open the English article.
4. Confirm all evidence appendix links work.
5. Confirm the article does not claim comfort.
6. Confirm the article does not claim collapse.
7. Confirm the article does not claim direct trade-down as proven.
8. Confirm Spanish is first and English is linked as parallel.
