# Claims audit

This page summarizes which claims are allowed, which remain unproven, and which are rejected inference boundaries.

## Visual

![Claim status counts](../assets/claim_status_counts_en.svg)

## Core rule

**Pressure is not behavior.**

Evidence can show pressure without automatically proving trade-down. It can also show macro resilience without proving comfort across all households.

## Main statuses

- `SUPPORTED_FORMAL_CHANNEL_ONLY`: supported only for the formal channel.
- `SUPPORTED_FOR_CONTEXT_ONLY`: useful as context, not closure proof.
- `NOT_PROVEN`: remains unproven.
- `REJECTED_BOUNDARY`: rejected inference boundary.

## Active limits

- Formal cushioning does not generalize to all households.
- Current labor context does not prove current income/remuneration.
- 2025 context does not prove 2026 behavior.
- Price-only evidence does not prove trade-down.
- Absence in tested sources does not prove global absence.
